# 🧠 JARVIS Web — Локальный ИИ-помощник с футуристическим интерфейсом

**JARVIS Web** — это веб-приложение, которое превращает вашу локальную Ollama в полноценного ИИ-помощника с голографическим интерфейсом, базой знаний и автоматическим обновлением. Вдохновлённый Железным человеком, он идеально подходит для задач программирования, кибербезопасности и работы с конфиденциальными данными — всё остаётся на вашем компьютере.

![Интерфейс JARVIS](screenshot.png) *(вставьте скриншот, если есть)*

## ✨ Особенности последней версии (v2.1)

- **🚀 Полностью локально** — никаких облаков, все данные только на вашем устройстве.
- **💬 Чат с ИИ** — поддержка любых моделей Ollama (Llama 3, CodeLlama, Qwen, Mistral и др.).
- **📚 База знаний (RAG)** — загружайте документы (PDF, TXT, MD, код) и задавайте вопросы по их содержимому.
- **🔄 Автообновление базы знаний** — раз в сутки система сама проверяет изменения в файлах и переиндексирует их.
- **🎨 Футуристический интерфейс** — анимированная сетка, неоновые панели, глитч-эффекты, адаптивный дизайн.
- **⚡ Быстрый WebSocket** — потоковая передача ответов, как в ChatGPT.
- **🔧 Улучшенная стабильность** — исправлена проблема с отображением ответов, отключена телеметрия ChromaDB.
- **🛠️ Удобные настройки** — выбор модели, температура, max tokens, системный промпт прямо из интерфейса.
- **🔒 Безопасность** — всё работает локально, никаких утечек данных.

## 📋 Системные требования

- **ОС**: Linux, macOS, Windows (с поддержкой WSL2 рекомендуется)
- **Python**: 3.10 или выше
- **Память**: от 8 ГБ (для моделей 7B параметров), для лёгких моделей (3B) достаточно 4 ГБ
- **Место на диске**: от 5 ГБ для моделей + ваши документы
- **Ollama** (устанавливается отдельно)

## 🚀 Быстрая установка

### 1. Установите Ollama

Перейдите на [ollama.ai](https://ollama.ai) и скачайте версию для вашей ОС, либо выполните в терминале:

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

Запустите сервер (обычно он стартует автоматически после установки):

```bash
ollama serve
```

Скачайте хотя бы одну модель, например, лёгкую и быструю:

```bash
ollama pull llama3.2:3b
```

Для программирования можно взять `codellama:7b` или `qwen2.5:7b`.

### 2. Клонируйте репозиторий (или создайте папку с файлами)

```bash
git clone https://github.com/yourusername/jarvis-web.git
cd jarvis-web
```

Если у вас нет git, просто создайте папку `jarvis-web` и поместите в неё файлы `app.py` и `requirements.txt` (см. ниже).

### 3. Создайте виртуальное окружение и активируйте его

```bash
python -m venv venv
source venv/bin/activate        # Linux/macOS
# или venv\Scripts\activate     # Windows
```

### 4. Установите зависимости

Создайте файл `requirements.txt` со следующим содержимым:

```txt
fastapi==0.115.0
uvicorn==0.30.1
httpx==0.27.0
jinja2==3.1.4
python-multipart==0.0.9
aiofiles==23.2.1
apscheduler==3.10.4
chromadb==0.5.0
sentence-transformers==3.0.0
PyPDF2==3.0.1
numpy==1.26.4
```

Затем выполните:

```bash
pip install -r requirements.txt
```

### 5. Подготовьте базу знаний (опционально)

Создайте папку `knowledge_base` в корне проекта и положите туда любые документы (поддерживаются `.txt`, `.md`, `.pdf`, `.py`, `.js`, `.html`, `.css`, `.json`).

Пример:

```bash
mkdir knowledge_base
cp ~/documents/*.pdf knowledge_base/
```

### 6. Запустите приложение

```bash
python app.py
```

Вы увидите сообщение:

```
🔄 Первоначальное обновление базы знаний...
🔍 Найдено изменений: 0 новых/изменённых, 0 удалённых
✅ Ollama доступна, найдено моделей: 4
🚀 Запуск JARVIS Web на http://localhost:8001
📁 База знаний: ./knowledge_base
🔄 Автообновление каждые 24 часов
📊 Использование БЗ: True
```

Откройте браузер и перейдите по адресу **http://localhost:8001**

## 🎮 Использование

### Основной экран

- **Левая панель** — управление: выбор модели, температура, max tokens, системный промпт, кнопки нового чата и обновления базы знаний.
- **Центральная область** — история чата.
- **Нижняя панель** — поле ввода сообщения и кнопка отправки.

### Как задать вопрос

1. Выберите модель в левом меню (список автоматически загружается из Ollama).
2. Настройте параметры при необходимости.
3. Введите сообщение и нажмите Enter или кнопку «ОТПРАВИТЬ».
4. Ответ будет выводиться потоково (как в ChatGPT). Индикатор печати показывается, пока идёт генерация.

### Работа с базой знаний

- Система автоматически использует найденные документы для ответа (RAG).
- Чтобы вручную обновить индекс после добавления/изменения файлов, нажмите кнопку **«🔄 ОБНОВИТЬ ЗНАНИЯ»**.
- Автообновление происходит каждые 24 часа (можно изменить в настройках, переменная `UPDATE_INTERVAL_HOURS`).

### Смена порта

По умолчанию используется порт **8001**. Если он занят, откройте `app.py` и в самом конце найдите строку:

```python
PORT = 8001
```

Замените `8001` на любой другой (например, `8002`) и перезапустите приложение.

## 🔧 Настройка под себя

### Изменение модели по умолчанию

В начале файла `app.py` есть переменная `DEFAULT_MODEL`. Укажите название модели, которую вы чаще всего используете:

```python
DEFAULT_MODEL = "llama3.2:3b"
```

### Интервал автообновления

Переменная `UPDATE_INTERVAL_HOURS` отвечает за частоту переиндексации базы знаний. По умолчанию 24 часа. Можно поставить меньше, например `1` для тестирования.

### Отключение базы знаний

Если вы хотите временно отключить использование базы знаний (например, для отладки), измените флаг `USE_KNOWLEDGE_BASE` на `False`:

```python
USE_KNOWLEDGE_BASE = False
```

### Параметры векторизации

В классе `KnowledgeBase` можно изменить размер чанка (`chunk_size`) и overlap в методе `_chunk_text`. По умолчанию 500 токенов с перекрытием 50.

## 🐛 Возможные проблемы и их решение

### Ошибка `Address already in use`

Порт 8001 уже занят. Смените порт, как описано выше.

### Не загружается список моделей

- Проверьте, запущена ли Ollama (`ollama serve`).
- Убедитесь, что есть хотя бы одна скачанная модель (`ollama list`).

### Долгие ответы

- Используйте более лёгкую модель, например `llama3.2:3b`.
- Уменьшите `max_tokens` в интерфейсе.
- Отключите использование базы знаний, если она не нужна (установите `USE_KNOWLEDGE_BASE = False`).

### Проблемы с русским языком в ответах

Убедитесь, что вы используете модель, поддерживающую русский язык (например, `llama3.2:3b`, `qwen2.5:7b`, `mistral`). При необходимости укажите в системном промпте: «Отвечай на русском языке».

### ChromaDB ругается на NumPy

Убедитесь, что установлена версия `numpy==1.26.4` (указана в `requirements.txt`). Переустановите при необходимости:

```bash
pip uninstall numpy -y
pip install numpy==1.26.4
```

### Нет ответа в чате (белый экран или пустота)

- Проверьте консоль браузера (F12) на наличие ошибок JavaScript.
- Убедитесь, что WebSocket подключился (статус «🟢 ONLINE»).
- В терминале сервера должны появляться логи `📨 Запрос...` и `✅ Ответ отправлен`. Если их нет, значит проблема на стороне клиента или в сети.

### Предупреждения `Failed to send telemetry event`

Это нормально, телеметрия ChromaDB отключена, но библиотека всё ещё пытается её отправить. Они не влияют на работу.
